# User Comment Journal

更新日: 2026-02-22

## 目的

このメモは、ユーザーコメントの変遷を後から振り返るための記録。
「最初に何を作りたいと言っていたか」と「途中で何が増えた/変わったか」を残す。

## 最初のコメント（出発点）

- iPhoneで背面2タップをトリガーに、現在表示中画面のスクリーンショットをAI解析したい
- 主な抽出対象は以下
  - 何のアプリなのか
  - 表示されている時間情報（カウントダウン、利用可能時刻など）
- 将来的に記憶系も入れたいが、まずは上記を優先
- 解析先は設定で切替したい
  - 端末内LLM
  - ローカルLLM
  - クラウドAPI
- スクリーンショット保存有無も設定で切替したい

## コメントの変遷（時系列）

1. 実装方式の整理
- Flutter単体ではなく、OS制約に合わせて起動導線を設計する方針に調整
- 最終的に「Flutterで進める」方針に確定

2. クロスプラットフォーム視点の追加
- iPhone/Androidを同じ体験に近づけたい意図を確認
- iPhoneは背面タップ割当、Androidは代替トリガーで統一導線にする方向

3. Ingressグリフハック用途の追加
- 図形を記憶・解析し、通知から素早く確認したい要望を追加
- 外字の直接表示は崩れやすいため、通知ではラベル+意味+画像添付を重視

4. AI利用制限の可視化ニーズ追加
- 「何時まで使えないか」を分かりやすく管理したい要望を追加
- UTC/JSTなど時刻表記の違いを吸収し、ローカル時間で表示したい
- 画面特徴を学習し、同種画面の制限判定精度を上げたい

5. 共有運用ニーズの追加
- 特定ユーザー依存ではなく、誰かが作った設定を共有して使える形を希望
- LINE等で共有できるよう、`Game Pack` 的な配布モデルを希望
- 自分の設定をエクスポートして配れる機能も希望

6. フォーマット設計方針の明確化
- 自由レイアウトではなく、一般的で決まった形式を希望
- `list / bar / countdown / checklist / table` の固定WidgetType方針を採用

7. ホーム画面体験の明確化
- 起動直後に管理リストを見たい
- 「解除まであと何分」「達成率」「今日あと何回必要か」を即確認したい

8. 実装簡素化の合意
- よりシンプルな実現として「ショートカット主体 + アプリ表示中心」を採用
- 重い拡張は後続に回す段階導入に合意

## 各段階でのアシスタント反応（概略）

1. 初期要件への反応
- iOS制約を明確化し、「背面タップ -> ショートカット連携」を前提に提案
- まずはMVPとして「アプリ判定 + 時間抽出」に集中する方針を提示

2. Flutter採用時の反応
- FlutterでUI/設定/結果管理を担当し、OS依存部分はネイティブ連携で補完する方針を提示
- 新規プロジェクト作成とMVP土台実装を実施

3. 機能拡張要望への反応
- グリフ記憶、制限可視化、共有運用の要望を仕様へ順次反映
- 安全面の懸念に対しては「固定JSON schema」「危険操作禁止」の制約を提案

4. 配布/署名の相談への反応
- サーバー前提ではなく、個人配布でも成立する設計を提示
- 実行コードを許さないデータ駆動方式を優先する方針を提示

5. 表示方式の相談への反応
- 自由レイアウトではなく、固定WidgetTypeでのプリセット表示を提案
- 汎用性と実装容易性のバランスで「内製固定JSON + 将来互換拡張」を提案

6. 簡素化の相談への反応
- 実装負荷を下げるため「ショートカット主体 + アプリ表示中心」の最小構成を提案

## 現時点の要約
